# About the Documentation

These docs follow [Diataxis](https://diataxis.fr/), but the practical goal is simpler than the framework vocabulary:

- [Getting Started](getting_started.md) shows how to use the template,
- [Maintain the Template](maintain_template.md) covers repository maintenance tasks,
- [Configuration](configuration.md) is the factual lookup page,
- [Features and Decisions](design_decisions.md) explains the rationale behind the defaults.

The documentation is written in Markdown with [MyST](https://myst-parser.readthedocs.io/) and built with [Sphinx](https://www.sphinx-doc.org/).
That keeps it aligned with the defaults generated by the template itself.

Environment variables should be defined once in [Configuration](configuration.md) and referenced from other chapters with glossary links such as {term}`GH_TOKEN`, {term}`GITHUB_TOKEN`, and {term}`DEMO_REPO_TOKEN`.

Local build shortcuts:

```bash
make docs
make docs-open
```

Equivalent direct command:

```bash
uv run --group docs sphinx-build docs docs/_build/html -b html -W
```

GitHub Actions validates docs in pull requests and publishes the canonical site from `main`, while documentation pull requests can also get preview deployments under `/_preview/pr-<N>/`.
